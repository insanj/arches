üêö Arches is a powerful webapp that scrapes and aggregates your Minecraft save data, allowing you to pool your player, inventory, and location information into new data structures. All features are accessible using the REST API which powers this webapp. The three core components of Arches are:

  1. Extract

    Drag and drop your Minecraft save data from ~/Library/Application Support/minecraft or %AppData%\.minecraft. Arches will parse and extract player, inventory, and location data from the save file (accessible using its NodeJS API).

  2. Transform

    Arches translates the extracted Minecraft save data into Arches NodeJS classes which can be represented visually in the Arches webapp. These items can also be exported in their own JSON format instead of the original Minecraft save format from the first step.

  3. Load

    A PostgreSQL database is loaded from the Arches NodeJS objects which are efficiently fit into relational models. The resulting database is stored in the location of the NodeJS server and will be updated every time a new save file is used in the Extract step.

    Once new records are created, the database view can be filtered and sorted using PostgreSQL. All webapp features are also easy to use without a UI using the REST API directly.

    All together, this allows for the tracking of journies - representations of change in player, location, and inventory location over time (and how each may relate to each other).
